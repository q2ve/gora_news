Я перекрыл интерфейсами датаклассы базы данных, чтобы была возможность
при необходимости легко заменить базу данных на другую, которая может
потребовать иную структуру своих классов с данными.

Например, хотим заменить Realm на Room.
Realm требует primary key в виде строки, свою аннотацию для него и
наследование класса от RealmObject.
Для Room нужен primary key типа Long со своей аннотацией и еще
аннотация для всего класса с данными.
Таким образом, мы не сможем работать с одними и теми же объектами
при замене базы данных.
А использоваться эти объекты могут по всему приложению.
